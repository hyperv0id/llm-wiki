# Heterogeneous Covariates (异构协变量)

## 定义

**异构协变量**是指与目标时间序列形式不同的外部变量，无法被传统的时间序列基础模型 (TSFMs) 直接处理，是 UniCA 框架要解决的核心问题[^src-unca]。

### 形式化定义

给定目标时间序列 $Y \in \mathbb{R}^{T \times 1}$，异构协变量 $X^{(het)}$ 是满足以下条件的外部变量：

1. **数据类型不同**：$X^{(het)}$ 的数据类型与 $Y$ 不同
2. **维度不匹配**：$X^{(het)}$ 的特征维度通常远高于或远低于 $Y$
3. **语义异构**：$X^{(het)}$ 携带的 信息与数值型时间序列在语义上不同

## 异构协变量的分类

### 类型 1：分类协变量 (Categorical Covariates)

**定义**：取离散值的协变量，表示某种类别或属性。

**常见示例**：

| 类别 | 示例 | 特点 |
|------|------|------|
| **实体标识** | 商品ID、门店ID、用户ID | 数量可能非常庞大 |
| **地理属性** | 城市、区域、门店类型 | 通常有层次结构 |
| **时间标记** | 星期几、月份、季度、节假日 | 周期性模式 |
| **事件类型** | 促销、节日、突发事件 | 影响显著但不频繁 |
| **产品属性** | 品牌、类别、颜色 | 影响需求模式 |

**处理挑战**：
- 类别数量可能非常大（数百万商品ID）
- 类别之间可能存在层级关系
- 某些类别在训练数据中可能很少出现

**传统处理方法**：
1. **One-Hot 编码**：类别数量少时可行，但维度爆炸
2. **整数编码**：简单但引入虚假的顺序关系
3. **Embedding**：将离散类别映射到低维连续空间（NLP 风格）

**UniCA 方案**：使用 embedding 层，然后通过同质化模块转换为时间序列表示[^src-unca]。

### 类型 2：多模态协变量 (Multimodal Covariates)

**定义**：非数值、非分类的高维数据，以图像、文本、音频等形式存在。

#### 2.1 图像协变量 (Image Covariates)

**典型应用场景**：

| 场景 | 图像类型 | 预测目标 |
|------|---------|----------|
| 太阳能预测 | 卫星云图 | 发电量 |
| 天气预测 | 气象卫星图 | 温度/降雨 |
| 交通预测 | 道路摄像头 | 车流量 |
| 零售预测 | 货架照片 | 销量 |
| 医疗预测 | 医学影像 | 健康指标 |

**处理挑战**：
- 高维像素数据（64×64×4 = 16,384 维）
- 空间结构需要提取
- 与时间序列的时间维度对齐

**UniCA 方案**：
- 使用轻量级 4 层 CNN 提取空间特征
- 将空间特征投影为时间序列表示[^src-unca]

#### 2.2 文本协变量 (Text Covariates)

**典型应用场景**：

| 场景 | 文本类型 | 预测目标 |
|------|---------|----------|
| 金融预测 | 财经新闻、财报 | 股票价格 |
| 能源预测 | 天气报告 | 电力需求 |
| 零售预测 | 商品描述、评论 | 销量 |
| 医疗预测 | 临床笔记 | 患者状态 |

**处理挑战**：
- 文本语义丰富但难以量化
- 文本长度可变
- 需要理解文本与时间序列的关系

**UniCA 方案**：
- 使用 GIST 编码器提取文本特征
- 转换为时间序列表示[^src-unca]

## 与同质协变量的对比

### 同质协变量 (Homogeneous Covariates)

**定义**：与目标时间序列形式相同的协变量，本质上也是实值数值时间序列。

**示例**：
- 预测某门店销量时，使用其他门店的销量作为协变量
- 预测电力需求时，使用温度、湿度等气象数据
- 预测股价时，使用其他相关股票的价格

**特点**：
- 可以直接与目标序列一起处理
- 现有 TSFMs（如 Moirai）有限支持
- 不需要复杂的模态转换

### 对比表格

| 特性 | 同质协变量 | 异构协变量 |
|------|-----------|-----------|
| **数据形式** | 实值数值序列 | 分类/图像/文本 |
| **维度** | 与目标相同 | 远高于或低于目标 |
| **处理复杂度** | 低 | 高 |
| **现有 TSFM 支持** | 部分支持 | 不支持 |
| **信息密度** | 稀疏 | 稠密 |
| **模态转换** | 无需 | 必须 |

### 数学表示

**同质协变量**：
$$C^{(hom)}_{1:T+H} \in \mathbb{R}^{(T+H) \times M_{hom}}$$

**异构协变量**：
- 分类：$c_{cat} \in \{1, 2, ..., K\}$
- 图像：$I_{t} \in \mathbb{R}^{H \times W \times C}$
- 文本：$w_{1:L} = [w_1, w_2, ..., w_L]$

**UniCA 的解决方案**：将两者都转换为 $C^{(het)}_{1:T+H} \in \mathbb{R}^{(T+H) \times d_{het}}$[^src-unca]

## 问题背景

### 为什么异构协变量难以处理？

1. **架构不匹配**：TSFMs 的Tokenizer 期望数值型时间序列输入
2. **预训练差距**：预训练数据中通常不包含异构协变量
3. **模态鸿沟**：图像/文本的语义空间与数值时间序列空间完全不同

### 实际应用中的异构协变量

现代时间序列预测任务越来越多地涉及异构协变量：

**零售预测**：
- 目标：商品销量
- 异构协变量：商品描述（文本）、商品图片（图像）、商品类别（分类）

**能源预测**：
- 目标：电力需求
- 异构协变量：天气预报（文本）、卫星云图（图像）、节假日（分类）

**金融预测**：
- 目标：股票价格
- 异构协变量：财经新闻（文本）、公司财报（文本）、K线图（图像）

### 现有方法的局限性

| 方法 | 同质支持 | 异构支持 | 训练方式 |
|------|---------|---------|----------|
| DeepAR | ✓ | ✗ | 从零训练 |
| TFT | ✓ | ✗ | 从零训练 |
| N-BEATSx | ✓ | ✗ | 从零训练 |
| TimesFM | ✗ | ✗ | 预训练 |
| Chronos | ✗ | ✗ | 预训练 |
| Moirai | 有限 | ✗ | 预训练 |
| **UniCA** | ✓ | ✓ | 适配器 |

> **关键问题**：虽然专用模型可以处理异构协变量，但它们缺乏预训练 TSFMs 的泛化能力。UniCA 旨在**兼得两全**：保留 TSFMs 的泛化能力，同时支持异构协变量[^src-unca]。

## 解决思路：协变量同质化

UniCA 提出的核心解决方案是**协变量同质化 (Covariate Homogenization)**：

1. **模态专用编码**：使用专用编码器提取特征
2. **线性投影**：将特征映射到统一的同质时间序列空间
3. **统一处理**：所有协变量作为同质时间序列处理

详细过程见 [[covariate-homogenization]]。
