# Karpathy: LLM Wiki

LLM으로 개인 지식 베이스를 구축하는 패턴을 제안하는 글.

## 핵심 주장

기존 RAG 방식(NotebookLM, ChatGPT 파일 업로드 등)은 매 질문마다 지식을 처음부터 재발견한다. **축적이 없다.** 이 패턴은 다르다 — LLM이 소스를 읽고 **지속적인 위키를 점진적으로 구축·유지**한다. 지식은 한 번 컴파일되고 계속 갱신된다.

> "The wiki is a persistent, compounding artifact."

## 3계층 아키텍처

| 계층 | 역할 | 소유자 |
|------|------|--------|
| **Raw Sources** | 원본 문서 (불변) | 인간 |
| **Wiki** | 마크다운 페이지 (요약, 엔티티, 컨셉) | LLM |
| **Schema** | 위키 구조·워크플로 정의 (CLAUDE.md 등) | 인간+LLM 공동 진화 |

## 3가지 오퍼레이션

- **[[ingest]]** — 새 소스 → 요약 페이지 생성, 기존 페이지 갱신, 모순 플래그, 인덱스·로그 업데이트. 소스 하나가 10-15개 페이지를 건드릴 수 있음.
- **[[query]]** — 위키 검색 → 답변 합성 → 좋은 답변은 위키에 다시 파일링.
- **[[lint]]** — 건강 검진. 모순, 고아 페이지, 누락된 교차참조, 데이터 갭 탐지.

## 활용 사례

- 개인 자기개선 추적, 연구 딥다이브, 독서 컴패니언 위키, 비즈니스 내부 위키, 경쟁 분석, 여행 계획 등.
- 핵심은 **시간에 걸쳐 지식이 쌓이는 모든 도메인**에 적용 가능.

## 도구

- **[[obsidian]]** — 위키 뷰어 겸 IDE. 그래프 뷰, 백링크, [[dataview]] 플러그인.
- **[[qmd]]** — 마크다운 로컬 검색 엔진 (BM25 + 벡터 하이브리드).
- Obsidian Web Clipper — 웹 기사를 마크다운으로 클리핑.

## 왜 작동하는가

지식 베이스 유지의 어려운 부분은 독서나 사고가 아니라 **북키핑**이다. 교차참조 갱신, 요약 최신화, 모순 추적 — 인간은 이 유지보수 부담 때문에 위키를 포기한다. LLM은 지루해하지 않고, 교차참조를 잊지 않으며, 한 번에 15개 파일을 건드릴 수 있다.

> Vannevar Bush의 Memex(1945) 비전과 정신적으로 연결됨 — 개인화된, 큐레이션된 지식 저장소.
